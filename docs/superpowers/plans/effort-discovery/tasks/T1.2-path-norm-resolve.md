# T1.2 — Path normalization + resolveActiveEffort

**Spec:** §6.3 (F6) + §6.2 (F11, F3). **Wave:** 1b (∥ T1.4a — disjoint files). **Depends on:** T1.1
(`effortBirthState`, merged). **Model:** capable.

## Objective

1. **Path-norm hygiene (§6.3):** make forward-slash paths the default by construction and give discovery a
   single case-folding comparison primitive.
2. **`resolveActiveEffort(cwd)` (§6.2):** the additive discovery wrapper that resolves the active effort by
   up-walk first, then a down-scan of born efforts under `.reasonable-efforts/` — so a nested effort is
   visible and a config-less stray is never adopted. This is the fix for the motivating incident's discovery
   hole. It is only ADDED here; T1.5 wires it into SessionStart.

## Scope

- **Edit:** `lib/effort.mjs` — `rootFromArgv` returns `norm(resolve(...))`; **hoist** `foldPath`/`samePath`/
  `underPath` here as **exported**; add `resolveActiveEffort`.
- **Edit:** `lib/reconcile.mjs` — remove the module-local `foldPath`/`samePath`/`underPath` (lines ~80-82),
  import them from `effort.mjs` instead. Behavior identical (do NOT change the win32-lowercase semantics).

  `conclude.mjs`, `docs/*`, `wo-status.mjs`, `ledger.mjs`.

## Interface (from shared/interfaces.md §T1.2 — use exactly)

```js
export function resolveActiveEffort(cwd)  // -> { kind, root?, roots?, strays? }
//   { kind:'resolved', root }         — exactly one born effort
//   { kind:'none', strays:[...] }     — no born effort; strays = config-less .reasonable dirs found
//   { kind:'multiple', roots:[...] }  — >1 born effort (NORMAL for parallel efforts)
```
Plus: `foldPath`, `samePath`, `underPath` become exported from `effort.mjs` (same win32-lowercasing they had
in reconcile); `rootFromArgv` returns `norm(resolve(...))`.

## Positive constraints (DO)

- **rootFromArgv:** wrap the explicit-`--root` and the `findEffortRoot` return in `norm(resolve(...))` so the
  path is absolute + forward-slash at the source. Keep the `--root` precedence + `argvWithoutRoot` behavior.
- **Hoist** `foldPath = p => process.platform==='win32' ? String(p).toLowerCase() : String(p)`,
  `samePath = (a,b) => foldPath(a)===foldPath(b)`, `underPath = (base,p) => foldPath(p).startsWith(foldPath(base)+'/')`
  verbatim into `effort.mjs`, exported. In `reconcile.mjs` delete the local defs and import them. Confirm
  every reconcile caller still works (they pass already-`norm`'d args).
- **resolveActiveEffort** per the pinned algorithm: up-walk first (correct for effort-root/worktree cwds),
  then down-scan `repoRoot/.reasonable-efforts/*` children whose EXACT `join(child,'.reasonable','config.json')`
  exists, excluding BACKUP_EXCLUDE names, keeping only `effortBirthState(child).state !== 'absent'`, ∪
  `repoRoot/.reasonable` if born. `repoRoot` via `git rev-parse --show-toplevel` (use `gitTry`), fallback cwd.
  0/1/N → none/resolved/multiple. A `.reasonable` at depth ≠ 1 under `.reasonable-efforts/` → a loud entry in
  the result (a diagnostic field or a stray), never silently dropped. Use `samePath`/`underPath` for every
  path comparison; forward-slash-only output (assert no backslash on any emitted path).
- New paths flow through `norm`/`samePath`, never naive `===`/`startsWith`.

## Negative constraints (DO NOT)

- Do NOT replace/remove `findEffortRoot` or `rootFromArgv`'s up-walk — the ~19 callers depend on them;
  `resolveActiveEffort` is an ADDITIVE wrapper only.
- Do NOT change the win32-lowercase semantics of `foldPath` (it's the case-fold the fence T0.3 note wants
  eventually generalized — but here you only HOIST it unchanged; a broader fence case-fold is out of scope).
- Do NOT adopt a config-less `.reasonable` as an effort (it is a `stray`, never `resolved`).
- Do NOT wire resolveActiveEffort into session-start (T1.5). No new dependency; no version bump.

## Before you begin
The down-scan's `git rev-parse --show-toplevel` returns the *worktree's* own root inside a linked worktree —
§6.2 avoids this by trying the up-walk FIRST (correct in worktree/effort-root cwds) and only down-scanning from
the repo-root interactive context. Honor that ordering. If the depth-≠1 diagnostic or the strays shape is
ambiguous, state your choice in the report. Otherwise: test-first, implement, run the whole suite (watch the
reconcile tests), self-review, commit. No version bump.
