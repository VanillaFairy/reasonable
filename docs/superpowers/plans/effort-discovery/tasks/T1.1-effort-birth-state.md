# T1.1 — effortBirthState predicate + write the birth signature

**Spec:** §6.1 (F4) + Defect A. **Layer:** 1, foundation (Wave 1a, solo). **Depends on:** Layer 0 (merged).
**Model:** capable.

## Objective

Two things, one concern (a durable, unambiguous "is this a born effort?" signal):
1. **`effortBirthState(effortRoot)`** — one shared predicate both discovery and reconcile will call, so they
   cannot disagree on what counts as a born effort.
2. **Defect A fix:** make `develop` Step 0 actually WRITE the birth signature (`config.effort`). Today
   NOTHING writes it — the effort name lives only in `journal.effort`, `loadConfig` has no `effort` default,
   yet `conclude.mjs` already reads `loadConfig(...).effort`. Without the write, every real effort would read
   `missing-signature` → HALT once the predicate is wired into reconcile (later task). Fix the root now.

## Scope

- **Edit:** `lib/effort.mjs` — add `effortBirthState`.
- **Edit:** `skills/develop/SKILL.md` — Step 0 writes `config.effort` (the effort name).
- **Create:** `test/effort-birth-state.test.mjs` (or add to `test/effort.test.mjs` — your call; a dedicated
  file is cleaner).
- **Do NOT edit:** `reconcile.mjs`, `session-start.mjs`, `conclude.mjs`, `fence.mjs`, `docs/*`. (Reconcile
  wiring of the predicate is a LATER task; you only add the predicate + the write.)

## Interface (from shared/interfaces.md §T1.1 — use exactly)

```js
export function effortBirthState(effortRoot)  // -> { state, reason? }
//   'absent'            : no .reasonable/config.json
//   'corrupt'           : config.json exists but does not JSON-parse
//   'missing-signature' : parses but no non-empty cfg.effort
//   'ok'                : parses, has a non-empty cfg.effort
```

## Positive constraints (DO)

- Make `effortBirthState` **pure + sync, no git**. It MUST distinguish `corrupt` from `absent`, so do NOT
  route through `loadConfig` (which swallows a parse failure into defaults). Read `.reasonable/config.json`
  directly: `existsSync` → if absent return `{state:'absent'}`; else `JSON.parse` in a try/catch → parse
  failure returns `{state:'corrupt', reason}`; else check `typeof cfg.effort === 'string' && cfg.effort.trim()`
  → `ok` vs `missing-signature`. Use the file's existing helpers (`join`, `readFileSync`/`existsSync` are
  already imported or trivially available) — stay node-builtin-only.
- In `skills/develop/SKILL.md` Step 0, add `"effort"` to the `config.json` write — the effort NAME (the same
  human name Step 0/analysis derives the `effort/<name>` branch from; a non-empty slug). Word it so the
  agent writes a stable identifier. Keep the existing `runMode`/`tier` writes intact. `config.json` is
  fence-protected, so this is a develop-Step-0-only write (correct).
- Keep `effortBirthState`'s return shape exactly `{state, reason?}`.

## Negative constraints (DO NOT)

- Do NOT wire `effortBirthState` into reconcile/session-start/conclude yet (later tasks). This task only
  ADDS the exported predicate + the develop write + tests.
- Do NOT change `loadConfig`'s defaults (do not add an `effort` default — the signature must come from a
  real on-disk write, and a default would make `missing-signature` unreachable, defeating the point).
- Do NOT touch the ~19 `findEffortRoot`/`rootFromArgv` up-walk callers.
- No new dependency; no version bump.

## Tests (test-first)

- `effortBirthState` fixtures: no config dir → `absent`; `config.json` = `not json{` → `corrupt`;
  `{"runMode":"gated"}` (no effort) → `missing-signature`; `{"effort":"demo","runMode":"gated"}` → `ok`;
  `{"effort":"   "}` (whitespace) → `missing-signature` (non-empty means trimmed-non-empty).
- Build the fixtures with the standard mkdtemp + `.reasonable/` harness (see `test/effort.test.mjs`).
- (develop is a skill doc, not code — no test; just confirm the SKILL.md edit reads coherently.)
