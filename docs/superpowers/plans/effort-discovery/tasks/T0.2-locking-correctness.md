# T0.2 — Locking correctness: atomic append + mirror under one lock

**Spec:** §5.3 (F1a) + §5.4 (F1b), combined — they are one concern (the ledger lock's coverage). **Wave:**
0a. **Depends on:** nothing. **Model:** capable (concurrency correctness).

## Objective

Two related races, both because work happens **outside** the ledger lock:
1. **§5.4 attempt-slot race.** `append()` resolves the attempt number via `buildTree()` **before** acquiring
   the lock (`ledger.mjs` ~lines 291/321; the lock is taken later inside `appendJsonl`). Two concurrent
   appends for one node can stamp the **same** attempt slot. `test/ledger.test.mjs` (~lines 451-482) today
   *asserts this collision* as a "known limitation."
2. **§5.3 torn/stale mirror.** `writeMirror()` runs **outside** the lock with plain `writeFileSync`
   overwrites (`progress-map.mjs` ~lines 216-229), called by `append()` after the lock is already released.
   Concurrent appends tear or stale `progress.*`, and even the unsynchronized PostToolUse-hook caller can
   expose a half-written file.

Fix both: attempt-resolution + seq + append + mirror-regen happen **atomically under one hold** of the
ledger lock, and each mirror file is written via tmp-then-rename.

## Scope

- **Edit:** `lib/ledger.mjs` (move `buildTree()`/attempt resolution inside the locked region), `lib/effort.mjs`
  (`withLock`/`appendJsonl` — widen the hold, or expose a lock-reentrant path so `append` can do
  read-resolve-write-regen under one acquisition), `lib/progress-map.mjs` (`writeMirror` atomic write).
- **Edit (test):** `test/ledger.test.mjs` (the collision assertion inverts to distinct-attempts).
- **Do NOT edit:** `reconcile.mjs`, `wo-status.mjs`, `fence.mjs`, `docs/*`.

## Interface (from shared/interfaces.md §T0.2)

No public signature changes. Contract:
- `append()` does attempt-resolution + seq + file-append + mirror-regen under **one** lock hold.
- `writeMirror()` writes `progress.json`/`progress.md` via `*.tmp-<pid>` + `renameSync`.
- `regen:false` still suppresses the mirror. Uncontended behavior unchanged.
- **Observable change:** N concurrent same-node `node-dispatched` → **distinct** attempts.

## Positive constraints (DO)

- Prefer the smallest correct change to `withLock`/`appendJsonl` that lets `append()` perform the whole
  read-resolve-write-regen sequence under one acquisition. If you restructure, keep `withLock`'s
  exclusive-create + spin-retry + 5000ms deadline + `finally`-unlink semantics intact.
- Make `writeMirror` atomic: distinct tmp name per process (`progress.json.tmp-<pid>`), `renameSync` over
  the target. On failure, clean up the tmp file; keep the existing fail-open-on-bad-ledger-data behavior.
- Keep the mirror regen *inside* the lock hold for the locked `append()` path (so a reader never sees a
  mirror that disagrees with the just-appended ledger line). The PostToolUse-hook direct `writeMirror`
  caller (outside `append`) still benefits from the atomic tmp+rename even though it holds no lock.
- Update `test/ledger.test.mjs`'s concurrency assertion to require **distinct** attempts across concurrent
  same-node dispatches; keep the rest of the test's intent.

## Negative constraints (DO NOT)

- Do NOT hold the lock across anything slow or unbounded (no subprocess, no network) — only the
  in-memory resolve + the two file writes.
- Do NOT change the ledger event grammar or `append()`'s return shape (`{ok, event}` / `{ok:false,error}`;
  never throws).
- Do NOT introduce a dependency or a second lock file; reuse the one `${ledgerPath}.lock`.
- Do NOT break `regen:false`.

## Tests (test-first)

- Concurrency: spawn N subprocess appends of `node-dispatched` for one node (pattern already in
  `ledger.test.mjs` ~350-377/451-482) → assert **N distinct attempts** (1..N), no duplicate slot.
- Mirror atomicity: interleave appends (or simulate) and assert `progress.json` always parses (never a torn
  read); assert a `*.tmp-*` file is not left behind after a successful append.
- Uncontended: a single append still stamps attempt 1 and regenerates the mirror; `regen:false` skips it.

## Acceptance

- `node test/ledger.test.mjs` green (with the inverted assertion); whole suite green.
- No duplicate attempt slots under concurrency; no torn/stale mirror.
- `withLock` semantics (deadline, cleanup) preserved; no new deps.

## Context

This is pure engine plumbing under `progress-map`/`ledger`/`effort`; it does not touch reconcile or the
fold. It runs in parallel with T0.1 (disjoint files) and T0.3. The `withLock`/`appendJsonl` helpers are in

attempt math run unlocked, then `appendJsonl` briefly locks only the seq-read + file-append. You are
