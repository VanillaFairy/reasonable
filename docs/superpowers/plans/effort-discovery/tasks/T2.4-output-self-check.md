# T2.4 — the output self-check (the projection's adversarial verification)

**Spec:** §7.4 (who verifies the projection — a mechanical, non-LLM invariant that refuses-with-reason).
**Wave:** 2d (serial after T2.3). **Depends on:** T2.2 (`projectDirectives`), T2.3 (the `next-action` append),
all Layer 0/1. **Model:** most-capable (adversarial logic + a load-bearing guard refactor).

## Objective

Add the mechanical self-check that gates the T2.2 projection **before** it is persisted (T2.3's append): a
directive that would resurrect a dropped WO, re-dispatch a dead-ended one, open a retired slice, or LAND over
a non-empty frontier is **refused with a reason** and downgraded to a `DECIDE` (which never auto-executes — so
even in autonomous mode it escalates, §7.4). This is the verification-trio applied to the projection itself:
the projection *proposes*, the self-check *refutes*, a refused directive becomes a human decision.

  `lib/next-action.mjs` (add `selfCheckDirectives`), `lib/reconcile.mjs` (compute the self-check `context`, gate
  the projection through it before the T2.3 append, record refusals). Tests: `test/redispatch-guard.test.mjs`

  a reconcile-level resurrection test.
- **Do NOT edit:** `lib/route.mjs`, `lib/ledger.mjs`, `lib/progress-map.mjs`, docs (T2.doc). Do NOT change
  `projectDirectives` (T2.2) or the `next-action` render (T2.3) — you gate BETWEEN them.

## Correction F (pinned — a §7.4 spec clarification, same status as Corrections A–E)
§7.4 literally says "never DISPATCH/RUNNING a WO with an unresolved amendment/**node-downgraded**." A
`node-downgraded` is the **legitimate reopen** mechanism (D19: a crashed/lost-work WO is downgraded back to
dispatchable so it can be retried) — refusing to dispatch a downgraded WO would **wedge crash recovery**, the
opposite of intent. The real, mechanically-sound invariant is **drop-authoritative**: a WO with an unresolved
`amendment` drop (or a hash-matched dead-end/infeasible verdict) must never be resurrected even if its spec
file still exists on disk (robustness matrix **S12**). That set is exactly what the **redispatch-guard**
already computes. So the self-check refuses a **guard-flagged** DISPATCH/RUNNING; it does **NOT** refuse on
`node-downgraded`. State this in a code comment + your report.

## 2. `lib/next-action.mjs` — `selfCheckDirectives(directives, context)` (PURE)
```js
export function selfCheckDirectives(directives, context)
//   -> { directives: Directive[], refusals: [{ directive, reason }] }
```
Pure, mechanical, non-LLM. `context` (pre-digested by reconcile — keep this function I/O-free):
```js
context = {
  guardBlocked: { [woId]: { reason } },   // WOs the redispatch-guard flags (dead-end / amendment-drop)
  routeSlices: string[],                  // the ratified frontier (readRoute().slices ?? [])
  frontierNonEmpty: boolean,              // is any known WO not terminal? (reconcile already knows this)
}
```
Walk the projected `directives`; for each, apply the refusal rules and REPLACE a refused directive with
`{ kind:'DECIDE', workOrder?/slice?, detail:'<refusal reason>' }` (a DECIDE never auto-executes → escalates in
both modes), collecting `{directive, reason}` into `refusals`:
- a **DISPATCH** naming any WO in `context.guardBlocked` → refuse (resurrection / dead-end — S12). (A DISPATCH
  may carry several WOs; refuse the whole directive with the offending id(s) named, or split — pick the
  cleaner, pin it.)
- a **RUNNING** naming a guard-blocked WO → refuse (same).
- an **OPEN** whose slice is **not in `context.routeSlices`** (a retired slice) → refuse.
- a **LAND** while `context.frontierNonEmpty` → refuse.
- Everything else passes through unchanged. Order preserved. NOTE (Correction F): do NOT refuse on
  `node-downgraded` — a downgraded WO is legitimately re-dispatchable.

## 3. `lib/reconcile.mjs` — compute `context`, gate before the append
Between the projection (T2.2, `result.nextAction`) and the T2.3 `next-action` append, insert the gate:
1. Build `context`: for each WO that appears in a DISPATCH/RUNNING directive (or all known WOs — cheap),
   call `redispatchBlock(ledger, spec)` (you already read each WO spec in T2.2) → `guardBlocked`. Set
   `routeSlices = routeOrder ?? []` and `frontierNonEmpty = frontierOpen` (reconcile already computed
   `frontierOpen` for `lifecycle`).
2. `const { directives, refusals } = selfCheckDirectives(result.nextAction, context);`
   set `result.nextAction = directives`; for each refusal `notes.push('self-check refused <kind> — <reason>')`.
3. THEN the T2.3 append persists `result.nextAction` (now the self-checked set) — a refused resurrection is
   never persisted as a DISPATCH, only as a DECIDE. (The append line from T2.3 is unchanged; it just now
   carries the gated directives.)
