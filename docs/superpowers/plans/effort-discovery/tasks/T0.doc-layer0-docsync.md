# T0.doc — Layer 0 doc-sync + version bump

**Spec:** repo invariant 3 (grammar + parser change together) + the spec's build-order doc-sync obligation.
**Wave:** 0c (after T0.1–T0.5 merge). **Depends on:** T0.1–T0.5. **Model:** standard.

## Objective

Transcribe Layer 0's landed changes into the design corpus so the docs never lag the parser, and bump the
plugin version once for the layer.

## Scope

- **Edit:** `docs/artifacts.md` (the machine-parsed grammars Layer 0 changed), `docs/DESIGN.md` and
  `docs/architecture.md` (cross-refs), `.claude-plugin/plugin.json` + any other place the version string
  lives (`README.md` install snippet + footer `Version:` line — grep for the current version).
- **Do NOT edit:** any `lib/` code or task-owned files (this is docs + version only).

## Positive constraints (DO)

- In `docs/artifacts.md`: document the **journal's narrowed shape** (lane registry + program-counter
  pointers, **no per-WO `status`**), and the **ledger grammar additions** (`amendment`/`ratification`
  `drops:[{workOrder,supersededBy?}]` + `resolvesSeq`). Mark them machine-parsed (the `*` convention).
- In `docs/DESIGN.md`/`architecture.md`: update D3b (journal-writer scope — derived index minus WO status;
  status is now the ledger fold) and note that WO status is a ledger fold (§5.1). Keep section numbers
  stable (they are cited from code).
- Read each merged Layer-0 implementer's reported "grammar delta" and transcribe it exactly — do not invent
  shapes; match what landed in `ledger.mjs`/`reconcile.mjs`/`wo-status.mjs`.
- Bump the version per repo policy: **minor** (backward-compatible new capability — status-as-fold, drop
  vocabulary, locking correctness, fence hardening). Update every place the version string appears.

## Negative constraints (DO NOT)

- Do NOT bump major.
- Do NOT edit code or restate a grammar the parser doesn't actually enforce — docs describe what shipped.
- Do NOT renumber DESIGN sections.

## Acceptance

- `docs/artifacts.md` matches the shipped journal + ledger grammars; DESIGN/architecture cross-refs updated;
  version bumped consistently everywhere; whole suite still green (docs-only, so it should be).

## Context

This is the single writer on `artifacts.md`/`DESIGN.md` for Layer 0, kept as its own task so parallel
implementers never contend on those files. Do it only after all five Layer-0 tasks are merged and their
reported grammar deltas are in hand.
