## `lib/next-action.mjs` — `projectDirectives(state)` (PURE — no I/O, no git)

```js
export function projectDirectives(state)  // -> Directive[]   (ordered; a SET, not a scalar)

// Directive = { kind, slice?, workOrders?, workOrder?, detail? }
//   kind ∈ 'HALT' | 'AMBIGUOUS' | 'DECIDE' | 'RUNNING' | 'DISPATCH' | 'RETRO' | 'OPEN' | 'LAND' | 'CONCLUDE' | 'DONE'
```

`state` — everything `projectDirectives` needs, pre-digested by `reconcile()` so this function stays pure and
table-testable:
```js
state = {
  halt: boolean, haltReason: string|null, ambiguities: [{ haltReason, ... }],   // reconcile's own
  openInbox: [{ kind, breaking?, ... }],                                        // reconcile's openInbox
  lifecycle: 'active' | 'at-land-gate' | 'half-concluded',                       // reconcile's lifecycle
  routeOrder: string[] | null,                                                   // readRoute(...).route?.slices ?? null
  workOrders: [                                                                  // one per known WO (journal ∪ ledger-only)
    { id, slice, status, dependsOn: string[], terminal: boolean,
      blocked: boolean, canceled: boolean, running: boolean } ],
  slices: [ { id, woIds: string[], allDone: boolean, retroDone: boolean } ],     // in routeOrder, pre-digested
}
```

### Projection logic (§7.3) — GLOBAL first-match, then the per-effort SET

1. **Global (first match wins → a single-element set):**
   - `state.halt` with a multi-root/ambiguous reason → `[{kind:'AMBIGUOUS', detail: haltReason}]`
   - `state.halt` (any other reason) → `[{kind:'HALT', detail: haltReason}]`
   - a **breaking** `openInbox` item → `[{kind:'DECIDE', detail:'inbox: <kind>'}]`
   - (the reconcile halt-class is already folded into `state.halt`/`haltReason`.)
2. **else drive off `state.lifecycle`:**
   - `'at-land-gate'` → `[{kind:'LAND'}]`
   - `'half-concluded'` → `[{kind:'CONCLUDE'}]`
   - `'active'` → the **SET** (multiple directives, in this order):
     - every **blocked** WO → `{kind:'DECIDE', workOrder:id}` (an open node-failed — decide redispatch or drop)
     - every **running** WO → collect into one `{kind:'RUNNING', workOrders:[ids]}`
     - every **ready** WO grouped by slice → one `{kind:'DISPATCH', slice:S, workOrders:[ids]}` per slice with
       ready WOs, **in `routeOrder`**
     - a slice whose `allDone` && !`retroDone` → `{kind:'RETRO', slice:S}`
     - a slice `retroDone` whose successor in `routeOrder` has no planned WOs yet → `{kind:'OPEN', slice:next}`
     - if EVERY known WO is terminal and no open slice remains → `[{kind:'DONE'}]`

### The readiness predicate (Corrections C + D — pin exactly)
A WO is **ready** iff **all** of:
- its own `status` is not terminal: not `'done'`, not `'dropped'`, and **not canceled** (`canceled===false`);
- **every** id in its `dependsOn` is `done` — where "done" is the dependency WO's `status === 'done'` in
  `state.workOrders` (**Correction C:** the fold/`workOrderStatuses` value; `merged` already folds into `done`;
  there is NO `'green'` and no `'merged'` status). A dep that is missing, pending, running, blocked, dropped,
  or **canceled** (**Correction D:** a deliberately-canceled dep is terminal-abandoned — its output will never
  exist) leaves the WO **not ready**.
- **NOTE — the adversarial exclusions are T2.4, not here:** `projectDirectives` does NOT consult the
  redispatch-guard, `amendment` drops, dead-ends, or retired-slice/land-nonempty. It emits the directives the
  *state* supports; T2.4's `selfCheckDirectives` is the mechanical adversary that strips a DISPATCH the guard
  flags, refuses a drop-resurrection, etc. Keep this function free of guard/git/drop logic.

## `reconcile()` wiring — assemble `state`, call, attach (this is the impure part)

Placement: **after** the per-WO crash-recovery loop and all partition/lifecycle/halt computation — i.e. right
before the `result` literal is built (~line 624). Everything you need (`workOrderStatuses`,
`terminalWorkOrders`, `blockedWorkOrders`, `deadEnds`, `lifecycle`, `halt`/`haltReason`/`ambiguities`,
`openInbox`) is settled by then. Then:

1. **Read `route.json`:** `readRoute(effortRoot)` → `routeOrder = res.route?.slices ?? null`. Surface a present
   `res.diagnostic` as a `notes.push(...)` (a broken route degrades the frontier, never crashes reconcile).
2. **Enrich each WO with `dependsOn` + `verticalSlice` from its SPEC** (the journal registry does NOT carry
   them — `journal.workOrders[id]` is the lane registry). For each known WO id, read
   `.reasonable/work-orders/<id>.json` (via `readJson`); take `dependsOn` (default `[]` if a legacy spec lacks
   it — forward-compat) and `verticalSlice`. Missing spec → `dependsOn:[]`, `slice:` fall back to
   `journal.currentVerticalSlice` or `null`, and note it.
3. **Detect canceled-terminal via the TREE, not the fold.** `workOrderStatuses` folds `node-canceled` → the
   inert `'pending'` (wo-status.mjs), so it CANNOT tell a deliberately-canceled WO from a fresh one. The
   progress tree can: `buildTree(effortRoot)` maps `node-canceled` → a `'canceled'` node status. Use the tree
   (reconcile already imports/uses `buildTree`) to set each WO's `canceled` flag. (This is exactly the
   RESUME-HERE carry-forward: a canceled WO is terminal, never re-dispatchable.)
4. **Per-slice digest:** group WOs by `slice` (their spec `verticalSlice`), ordered by `routeOrder`; for each,
   `allDone` = every member WO `status==='done'`; **`retroDone`** = derive from `journal.currentVerticalSlice`
   + `routeOrder` position: a slice strictly BEFORE the current slice in `routeOrder` with all WOs done is
   retro-passed; the current slice is not retro-done while it is current. **There is no clean per-slice
   retro-done ledger event** — you are deriving it from journal position; PIN your exact rule in a code comment
   and your report, and cover it with a test. If you cannot derive it defensibly, emit the WO-level directives
   (DISPATCH/RUNNING/DECIDE) + the terminal LAND/CONCLUDE, and OMIT the RETRO/OPEN pair rather than guess —
   note it as a follow-up. (WO-level + LAND/CONCLUDE is the high-value core; RETRO/OPEN is the nicety.)
5. **Assemble `state`, call `projectDirectives(state)`, attach as `result.nextAction`** (an array). Do NOT
   append a ledger event, do NOT render — that is T2.3.
