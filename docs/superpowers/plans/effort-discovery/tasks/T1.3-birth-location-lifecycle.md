## Objective

Three related parts of "effort identity + state on disk," landed as **three separate commits**:
- **A — birth-location policy:** kill stray-root rebirth at its true source (birth location), so a run that
  loses its `--root` can't silently re, birth a repo-root `.reasonable/` next to a real nested effort.
- **B — lifecycle-state:** reconcile classifies the BORN effort's state (active / at-land-gate /
  half-concluded) so the briefing (T1.5) and nextAction (Layer 2) can act on it deterministically.
- **C — gitignore the abandoned archive:** the one-line fix the T1.4a review surfaced.

## Interface (from shared/interfaces.md §T1.3 — use exactly)

### Part A
- `export function assertNoAmbiguousBirth(repoRoot)` → `{ ambiguous:boolean, existing:[roots] }`. `ambiguous`
  true when `repoRoot/.reasonable-efforts/*/` holds ≥1 BORN effort (via `effortBirthState(child).state !== 'absent'`
  on the exact `<child>/.reasonable/config.json`, same exclusions as the T1.2 down-scan). fs read, no mutation.
- `develop` Step 0 (prose): before writing `config.json` at a bare repo-root cwd, if
  `assertNoAmbiguousBirth(repoRoot).ambiguous`, REFUSE the bare-birth and instruct an explicit nested
  `--root .reasonable-efforts/<name>/`.
- `fence.mjs` first-birth path (~422-423): when `findEffortRoot(tgt)` is null AND `.reasonable-efforts/` holds
  born efforts → `deny("possible stray birth — pass --root <nested>")` instead of the silent `process.exit(0)`.
  A truly plain repo (no `.reasonable-efforts/`) STILL fails open (do not break invariant #2).
- `reconcile.mjs`: an AMBIGUOUS bucket `{haltReason, evidence}` for **>1 `.reasonable`-bearing root reachable**
  (a scan / `resolveActiveEffort(...).kind==='multiple'`), joining `ambiguities[]` → HALT.

### Part B — reconcile `lifecycle` field
Deterministic predicates, cheapest first, for the states reconcile can see (it only runs on a live `.reasonable/`):
- `'active'` — the route/frontier has an open slice.
- `'at-land-gate'` — frontier empty AND `descendsFrom(effortBranch, base) === false` (work not yet landed to base) → the briefing will say LAND.
- `'half-concluded'` — `descendsFrom(effortBranch, base) === true`, still a live `.reasonable/`, no `.done-*` → the briefing will say CONCLUDE.
Use `git merge-base --is-ancestor <base> <effortBranch>` via `gitTry` for `descendsFrom`. Add `lifecycle` to
the reconcile result. Do NOT compute nextAction (T2.2 consumes `lifecycle`). The dir-name states
(`concluded`/`abandoned`/`stray`) are T1.5's scan — not here.

### Part C
- `skills/analysis/SKILL.md` (~line 106-107): add `.reasonable.abandoned-*/` to the planted `.gitignore` set
  beside `.reasonable.done-*/`.
