# T0.3 — Fence denies direct ledger writes for every role

**Spec:** §5.5 (F1c). **Wave:** 0a. **Depends on:** nothing. **Model:** standard.

## Objective

The fence trusts `role === null` (main session) and exempts it from all `.reasonable/` governance. So two
trusted parallel sessions can `Edit`/`echo >>` `.reasonable/ledger.jsonl` directly, bypassing the
`ledger.mjs` lock — a silent lost update with no detector. Close it: **a direct write to
`.reasonable/ledger.jsonl` is denied for every role, including the main session.** All writers go through
`ledger.mjs append()` and its lock. Main session keeps its trust for everything else in `.reasonable/`.

## Scope

- **Edit:** `lib/fence.mjs` (add the ledger-write denial ahead of the `role===null` exemptions).
- **Edit (test):** `test/fence.test.mjs` (add cases).
- **Do NOT edit:** any other lib, `ledger.mjs`, `docs/*`.

## Interface (from shared/interfaces.md §T0.3)

- Target `.reasonable/ledger.jsonl` via `Edit`/`Write`/shell append → **deny, all roles**, including
  `role===null`.
- All other `.reasonable/` writes by the main session → still allowed.
- Deny message names the file and points to the CLI, e.g.
  `direct writes to .reasonable/ledger.jsonl are forbidden — append only via lib/ledger.mjs (locked)`.

## Positive constraints (DO)

- Add the ledger-write denial so it fires **before** every `role===null` allow in the relevant paths.
  Recon points at the main-session exemptions: the structured-edit canonical branch (`fence.mjs` ~line 401),
  the Bash backstop (~line 286), and the in-effort no-lane main allow (~line 408). The denial must dominate
  all write channels that can reach `ledger.jsonl` (structured Edit/Write **and** shell `>>`/`tee`/`cp`).
- Detect the target by path, matching `.reasonable/ledger.jsonl` robustly (use existing path helpers —
  `norm`, the enforcement-path matching already in the file; `.reasonable/ledger.jsonl` is already in
  `enforcementPaths`). Prefer reusing the existing matcher over a fresh regex.
- Keep the deny shape identical to other fence denials (the `deny(...)` helper / JSON the tests parse).

## Negative constraints (DO NOT)

- Do NOT broaden the denial to other `.reasonable/` files — only `ledger.jsonl`. The main session must keep
  writing config/journal/route/etc. as it does today. (Journal has its own scribe fence; do not touch it.)
- Do NOT block the `ledger.mjs` CLI itself — the fence governs tool calls (Edit/Write/Bash), not the locked
  library path. A Bash call that runs `node .../ledger.mjs append ...` is allowed; a Bash call that does
  `echo ... >> .reasonable/ledger.jsonl` is denied.
- Do NOT change fail-open-outside-an-effort: with no effort reachable, everything is still allowed.
- Do NOT alter the enforcement-immutability rules or other categorical laws.

## Tests (test-first)

- Main session (`role===null`) `Write`/`Edit` to `.reasonable/ledger.jsonl` → **denied** (assert the deny
  JSON + reason).
- Main session shell append `echo x >> .reasonable/ledger.jsonl` (and a `tee`) → **denied**.
- Main session `Write` to `.reasonable/route.md` (or `config.json`) → **allowed** (unchanged).
- A `Bash` call invoking `node <plugin>/lib/ledger.mjs append ...` → **allowed** (not a direct write).
- Outside any effort (no `.reasonable/` reachable) → a write anywhere is allowed (fail-open preserved).

## Acceptance

- `node test/fence.test.mjs` green; whole suite green.
- Direct ledger writes denied for all roles including main; other `.reasonable/` main writes unaffected;
  fail-open-outside-effort intact.
