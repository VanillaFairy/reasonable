# T1.4a — The abandon command

**Spec:** §6.5 (F10), the command half. **Wave:** 1b (∥ T1.2 — disjoint files). **Depends on:** T1.1
(`config.effort` birth signature, merged). **Model:** standard.

## Objective

A first-class **`reasonable:abandon`** action so a walked-away effort drops out of the discovery scan the
same cheap way a concluded one does — by a dir rename + a ledger event — instead of lingering as a live
effort forever. It mirrors `conclude` exactly; only the archive suffix (`.reasonable.abandoned-<effort>`)
and the ledger event type (`abandoned`) differ. This is the COMMAND only — computing the *lifecycle state*
(Concluded/Abandoned/Active/…) in reconcile is T1.3, not this task.

## Scope

- **Create:** `lib/abandon.mjs` (clone of `lib/conclude.mjs`), `test/abandon.test.mjs`.
- **Edit:** `lib/ledger.mjs` — add `'abandoned'` to `EVENT_SCHEMAS` + `FAMILY_1_TYPES`.
- **Edit:** `lib/progress-map.mjs` — add `'abandoned'` to `EVENT_MAP` (a terminal note, mirroring `'concluded'`).
- **Create/edit:** a `reasonable:abandon` user-invocable skill/command, mirroring however `conclude` is
  surfaced (find it first — check `commands/`, `skills/`, `hooks/hooks.json`, and how `conclude.mjs` is
  invoked; mirror that surface exactly).
- **Do NOT edit:** `effort.mjs`, `reconcile.mjs` (T1.2/T1.3 own those), `conclude.mjs` (leave it as the
  template), `docs/*`.

## Interface (from shared/interfaces.md §T1.4a)

- `node abandon.mjs [path] [--root <p>]` — same CLI shape as conclude (top-level script, no export/guard).
- Appends `{type:'abandoned', effort}` via the ledger controller (regen-on; fatal if `!ok`).
- Archives `.reasonable/ → .reasonable.abandoned-<effort>/`.
- `'abandoned'` added to `EVENT_SCHEMAS` (required `[]`, like `'concluded'`), `FAMILY_1_TYPES`, `EVENT_MAP`.

## Positive constraints (DO)

- Clone `lib/conclude.mjs` structure verbatim, changing only: the archive dir suffix (`.reasonable.abandoned-<effort>`
  instead of `.reasonable.done-<effort>`), the ledger event type (`'abandoned'` instead of `'concluded'`),
  and the user-facing messages. Keep the SAME: `rootFromArgv`/`argvWithoutRoot` root resolution, the
  fail-open "nothing to abandon" no-effort exit, the archive-already-exists refusal, the `commitGate`
  (commit residual in-scope work, HALT if still dirty), the `renameSync` archival, and the tracked-deletion
  commit. `effort = loadConfig(...).effort || 'effort'` (T1.1 now writes `config.effort`, so this resolves).
- In `ledger.mjs`: add `'abandoned': { required: [] }` to `EVENT_SCHEMAS` next to `'concluded'`; add it to
  `FAMILY_1_TYPES`. In `progress-map.mjs` `EVENT_MAP`: add an `'abandoned'` entry mirroring `'concluded'`'s
  op (a terminal/annotation on the effort root — copy whatever `'concluded'` does).
- Surface `reasonable:abandon` the same way `conclude` is surfaced to the user (mirror the existing pattern
  — do not invent a new surface).

## Tests (test-first)

  `'abandoned'` ledger line, renames `.reasonable/ → .reasonable.abandoned-<effort>/`, exit 0; a dirty tree
  with in-scope work → commit-gate commits it then archives (or HALTs if it can't reach clean, matching
  conclude's contract — replicate conclude's test intent); archive-already-exists → refuse (exit 1);
  no-effort → fail-open exit 0.

- Confirm `progress-map` folds an `'abandoned'` event without a `[fold error]` (EVENT_MAP entry present).
