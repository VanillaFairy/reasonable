# T1.5 — Multi-effort briefing + reconcile S7 born-state HALT

**Spec:** §6.6 (F7) + §6.1 (F4, the S7 re-key). **Wave:** 1d (solo). **Depends on:** T1.2
(`resolveActiveEffort`), T1.3 (`lifecycle`) — both merged. **Model:** capable. **Last Layer-1 impl task.**

## Objective

Two parts, two commits:
- **A — the multi-effort SessionStart briefing (§6.6):** a fresh session in any state deterministically
  resolves *which* efforts exist and *what is left* in each — as a read, cheaply, without reconciling every
  effort. This is what makes the motivating incident's "which effort is active?" answerable at a glance.
- **B — reconcile S7 born-state HALT (§6.1):** close the hole where a born effort with a `missing-signature`
  (or corrupt) config silently proceeds instead of HALTing.

## Interface (from shared/interfaces.md §T1.5 — use exactly)

### Part A — session-start
Replace the raw `findEffortRoot(cwd)` discovery (`session-start.mjs:16`) with `resolveActiveEffort(cwd)`:
- `kind:'resolved'` → the CURRENT behavior for that one effort: `writeMirror` (fail-open) + `reconcile(root)`
  + `briefing(r)` (now includes `lifecycle`) + the "effort ACTIVE (mode/tier)" line.
- `kind:'multiple'` → CHEAP per-effort briefing, NO per-effort reconcile: for each root read
  `.reasonable/progress.json` (counts) + the last `.reasonable/ledger.jsonl` event `ts` for **staleness**
  (days-since; `Date` is allowed in this hook script) + the persisted `nextAction` **if present** (absent in
  Layer 1 → fall back to counts + a "reconcile on demand" note). Wrap EACH effort in its own `try/catch`
  (never one around the loop). Reconcile only the acted-on effort / on demand.
- `kind:'none'` → "no active effort"; surface `strays` + `diagnostics` as **debris + a cleanup note**, never
  adopted.
- Filter **concluded/abandoned** dirs (`.reasonable.done-*` / `.reasonable.abandoned-*`) by NAME — count them
  ("N parked/stale hidden"), never brief/scan.
- **Flag a born-but-bad config:** any resolved/listed effort whose `effortBirthState(root)` is `corrupt`/
  `missing-signature` → FLAG in the briefing (HALT-worthy, Part B), not silently proceed.

### Part B — reconcile S7
reconcile HALT keys on `effortBirthState`, not only `loadConfig`'s `runMode:null`: `effortBirthState(root)` ∈
`{corrupt, missing-signature}` → an AMBIGUOUS→HALT `{haltReason, evidence}` bucket. Keep the existing
runMode-absent HALT. (`corrupt` already halts incidentally via runMode-absent; `missing-signature` — has
`runMode`, no `effort` — currently PROCEEDS; that is the hole to close.)
